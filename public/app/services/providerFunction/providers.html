Services created with the Provider Function.

Important notes about the provider function:

  • Defined by $provide.provider('serviceName', callback)
  • Instead of using the app.config(function ($provide) { $provide.provider('serviceName', callback, Angular allows us to use the short version app.provider('serviceName', callback
  • The name of the provider will be serviceNameProvider.
  • Inside the callback, this.$get is equal to another function that is the one that defines the service.
  • The function that defines the service normally follows the module pattern: first we define variables and methods, then we return an object that will have those variables and methods as attributes.
  • The provider function is a little bit more complicated than the other ways to create services, but it allows you to configure the provider that creates the service.
  • The way we configure the underlying provider is by including getters and setters in the provider function.
