$resource for simple RESTful APIs.

Notes:

  • In order to use $resource, we need to load the angular-resource module and to add the module dependency (ngResource).
  • $resource is just an abstraction layer above $http, designed with the goal in mind of having a simple and quick way to manage restful apis.
  • The $resource documentation highlights two important concepts for this service: what is called a "class" var Books = $resource('api/books/:bookId') and what are called the "instances" of that class var book5 = Books.get({bookId: 5}) (this gets you the whole book object that has the id of 5).
  • Methods of a $resource class:
    1. get(): retrieve a particular item.
    2. save(): create a new item.
    3. query(): retrieve all items.
    4. remove() and delete() do the same: delete an item.
    5. You can define custom methods.
  • Methods of a $resource instance:
    1. Entity properties (title, author, etc).
    2. $promise: gives you access to the promise created when connecting with the server.
    3. $resolved: boolean that tells you if the promise has been resolved or not.
    4. $get, $save, $query, $remove/$delete.
    5. custom methods (prefixed by $).

Using $resource:

  1. Create a service using the factory function.
  2. Inject $resource.
  3. The only purpose of this service is to return a $resource class that we can manipulate later.
  4. The $resource class will have the following parameters:
    • url of the server api
    • the second parameter is where you set parameter defaults.
    • the third parameter allows you to set additional methods on top of the default methods of the $resource class.
