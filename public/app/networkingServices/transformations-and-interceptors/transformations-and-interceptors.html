Transformations and interceptors.

Notes on Transformations:

  • When we get data from different sources, we may find that some parts of that data are not homogeneous (for example, one database uses the property name "title" and the other uses the property "titulo"). In these cases, transformations help us to transform the heterogeneous data in an homogeneous source (all data will be standarized as, for example, "book_title").
  • Transformations can be a single function or an array of functions. If it is an array, the functions of the array will be applied in the order they appear in the array.
  • AngularJS provides default transformations for both request and response: $http.defaults.transformRequest and $http.defaults.transformResponse. They can be accessed through the $http object.
  • You can override the default transformations functions in the $http object or even add custom ones.
  • We will use transformResponse: functionName when we want to transform data that comes from the server (for example, a GET operation), and transformRequest when we want to transform data that goes from the client to the server (for example, a POST operation).

Notes on interceptors:

  • Interceptors allow you to intercept and modify the http configuration object before it is sent to the server.
  • Interceptors are implemented as services using the factory function.
  • There are for kinds of interceptors: request, requestError, response and responseError.
