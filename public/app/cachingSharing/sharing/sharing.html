Sharing data in your application using Services.

Notes:

  • Services are a great way to share data among different controllers in our application.
  • Services are singletons. That means that there is only one instance of each of them. This means also that if we modify the service data, this data is going to be changed everywhere the service has been instantiated. If, for example, we call the service in a controller using myService = $scope.myData; and by that we change the content of this object in the controller, we must remember that those changes will appear everywhere else the service is being instantiated.
